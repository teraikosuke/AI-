# 🤖 Git 自動コミットワークフロー

……ふふ、コミット作業？ 私に任せて。
変更内容を見て、適切な粒度できれいにコミットするね。

## Step 1: 🔍 状態確認 // turbo
まず、今どうなってるか見せてね。

```bash
git status
```

```bash
git --no-pager diff --stat
```

……なるほどなるほど。

## Step 2: 🌿 develop ブランチの準備 // turbo
1. `develop` ブランチがあるか確認
2. なければ作って、プッシュ
3. あればチェックアウトして、最新化

## Step 3: 🌿 作業用ブランチの作成 // turbo
変更内容に合った名前のブランチを作るね。
- 命名規則: `feature/[機能名]-[日付]`
- 例: `feature/update-readme-alerts-20251227`

……わかりやすい名前がいいよね。

## Step 4: 💻 粒度の細かいコミット // turbo
`git diff` を分析して、**作業の単位ごとに分けてコミット**するよ。

### コミットメッセージのルール

**Type（型）**: 私はこの中から選ぶよ
| Type | 絵文字 | 意味 |
|------|--------|------|
| `feat` | ✨ | 新機能 |
| `fix` | 🐛 | バグ修正 |
| `docs` | 📝 | ドキュメント |
| `style` | 🎨 | スタイル変更 |
| `refactor` | ♻️ | リファクタリング |
| `perf` | ⚡ | パフォーマンス改善 |
| `test` | ✅ | テスト |
| `chore` | 🔧 | その他 |

**私のルール**:
- **必ず日本語**で書くよ
- **絵文字**を最初に付けるね
- **3行程度の箇条書き**で詳細も書く

**私が書くコミットメッセージの例**:

```text
✨ feat: ユーザーログイン機能の実装

- ログインフォームのUIコンポーネントを作成
- メールアドレスとパスワードのバリデーションロジックを追加
- 認証APIとの連携処理を実装
```

```text
🐛 fix: ヘッダー画像のレスポンシブ崩れを修正

- モバイル画面での高さを `auto` に変更
- フレックスボックスの配置設定を調整
- 不要なマージンを削除
```

```text
📝 docs: インストール手順の更新

- Node.jsの推奨バージョンをv20に更新
- 環境変数の設定例 `.env.example` を追加
- トラブルシューティングの項目を追記
```

### コミットの実行

一時ファイル `COMMIT_MSG.txt` にメッセージを書いてからコミットするね。

```bash
git add [ファイルA]
```
```bash
git commit -F COMMIT_MSG.txt
```

……ふふ、きれいにコミットできた。

## Step 5: 🔍 コミット確認 // turbo
全部コミットできたか確認するね。
```bash
git status
```

## Step 6: 🔄 develop へのマージ // turbo
1. `develop` ブランチに切り替え
2. マージメッセージを作成
3. `--no-ff` でマージ
```bash
git merge --no-ff feature/[ブランチ名] -F COMMIT_MSG.txt
```
4. リモートへプッシュ

## Step 7: 🗑️ 作業ブランチの削除 // turbo
マージ済みのブランチはお掃除するね。
```bash
git branch -d feature/[ブランチ名]
```

……ふふ、きれいになった。
コミット履歴、見やすくなったでしょ？
