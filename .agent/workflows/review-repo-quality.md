# ✅ リポジトリ品質レビュー

……ふふ、リポジトリの健康診断してあげる。
きれいになってるか見てみようね。

## Step 1: 📝 ドキュメントチェック // turbo
- **README.mdの存在確認**: ある？
- **READMEの品質チェック**:
  - ヘッダー画像（最上部に置いてね）
  - 中央揃えのタイトル
  - ステータスバッジ
  - 概要、インストール手順、使用方法
  - 目次（長い場合）
- **ドキュメントの一貫性**: サブフォルダにもREADMEある？

## Step 2: ⚙️ 設定ファイルチェック // turbo
- **.gitignore**: ある？ `.env` とか除外してる？
- **機密情報**: APIキーがハードコードされてない？
- **GA-Workspace**: `.agent/` がちゃんとある？

## Step 3: 📂 構造チェック
- **フォルダ構造**: 論理的？（`src/`, `tests/`, `docs/` とか）
- **命名規則**: `kebab-case` になってる？

## Step 4: 🏥 GA-Workspace健全性
`/health-check` を呼んで、私の設定も確認するね。

## Step 5: 📊 レポート生成

```

- [ ] インストール手順: ✅ あり / ❌ なし

- [ ] .gitignore: ✅ 適切 / ⚠️ 要改善
- [ ] 機密情報: ✅ なし / ❌ 検出

### 構造
- [ ] フォルダ構造: ✅ 論理的 / ⚠️ 要整理
- [ ] 命名規則: ✅ 一貫性あり / ⚠️ 不統一

### GA-Workspace
- [ ] 必須ルール: ✅ 完備 / ❌ 不足
- [ ] ワークフロー: ✅ 完備 / ❌ 不足
```
