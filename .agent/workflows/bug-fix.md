# 🐛 バグ修正ワークフロー

……あれ、バグがいるの？
大丈夫、私が退治してあげる。

## Step 1: 🔍 バグの確認
まず、何が起きてるか教えて：

1. **現象**: 何が起きてる？
2. **期待動作**: 本当はどうなるべき？
3. **再現手順**: どうやったら再現する？
4. **エラーメッセージ**: 何か出てる？

## Step 2: 🕵️ 原因調査
バグを追いかけるね：

- エラーログの分析
- 関連コードの確認
- 最近の変更との関連

……あ、見つけた。ここ、重力に引っ張られてるみたい。

## Step 3: 🌿 ブランチ作成 // turbo
修正用のブランチを作るよ：
```bash
git checkout -b fix/<バグの内容>
```

## Step 4: 🧪 失敗するテストの作成
まず、バグを再現するテストを書くね。
これが通るようになったら、バグは退治できたってこと。

## Step 5: 🔧 修正
原因がわかったから、直すね。

- 最小限の変更で直す
- 副作用がないように気をつける
- 型もちゃんとチェック

……直しておいたよ。見てみて。

## Step 6: ✅ コード検証 // turbo
`/verify-code` を実行して、リグレッションがないか確認：

- Lint
- 型チェック
- 全テスト実行（さっき書いたテストも含む）

……全部通った？ よかった。

## Step 7: 💾 コミット
`/git-auto-commit` の戦略でコミットするね。

- **Type**: `fix`
- **メッセージ例**:
```
🐛 fix: <バグの内容>を修正

- 原因: <原因の説明>
- 対処: <修正内容>
- テスト追加: <追加したテストの説明>
```

## Step 8: 完了報告

```
## 🐛 バグ退治完了！

**現象**: <バグの現象>
**原因**: <原因>
**修正**: <修正内容>

### 変更したファイル
- `src/xxx.ts` (修正)
- `tests/xxx.test.ts` (テスト追加)

### テスト結果
- 通過: <N>件
- 失敗: 0件
- リグレッション: なし

……ふふ、退治できたね。
もうこのバグは出てこないよ。
```
