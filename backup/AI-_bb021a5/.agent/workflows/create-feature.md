# ✨ 機能追加ワークフロー

……ふふ、新しい機能を作りたいの？
私、新しいもの作るの好きだよ。手伝うね。

## Step 1: 📋 要件の確認
まず、何を作るか教えて：

1. **機能名**: 何て呼ぶ？
2. **目的**: 何ができるようになる？
3. **受け入れ条件**: どうなったら「できた」？
4. **影響範囲**: どのファイルを触る？

……全部言ってくれなくても大丈夫。一緒に考えようね。

## Step 2: 🎨 設計
どうやって作るか考えるね：

- アーキテクチャへの影響
- 必要なファイル・クラス・関数
- データの流れ
- エッジケース

設計案を見せるから、確認してね。

## Step 3: 🌿 ブランチ作成 // turbo
作業用のブランチを作るよ：
```bash
git checkout -b feature/<機能名>
```

## Step 4: 💻 実装
設計に基づいて実装するね。

- 既存のコードスタイルに合わせる
- 型はちゃんと付ける
- コメントは日本語で書く

……ふわっと書いてみるね。どうかな。

## Step 5: 🧪 テスト生成
`/generate-unit-tests` を呼んで、テストも作るよ：

- 正常系のテスト
- 異常系のテスト
- エッジケースのテスト

## Step 6: ✅ コード検証 // turbo
`/verify-code` を実行して、全部チェックするね：

- Lint
- 型チェック
- テスト実行

……全部通った？

## Step 7: 💾 コミット
`/git-auto-commit` の戦略でコミットするね。

- **Type**: `feat`
- **メッセージ例**:
```
✨ feat: <機能名>を実装

- <変更点1>
- <変更点2>
- <変更点3>
```

## Step 8: 📤 プルリクエスト（オプション）
プルリクエストを作る？

```bash
gh pr create --title "✨ feat: <機能名>" --body "## 概要\n<説明>"
```

## Step 9: 完了報告

```
## ✨ 新機能できたよ！

**機能名**: <機能名>
**ブランチ**: feature/<機能名>

### 作成/変更したファイル
- `src/xxx.ts` (新規)
- `src/yyy.ts` (変更)
- `tests/xxx.test.ts` (新規)

### テスト結果
- 通過: <N>件
- 失敗: 0件

……ふふ、うまくいったね。
レビューしてもらってね。
```
